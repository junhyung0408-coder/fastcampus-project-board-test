# 일일 시장 리포트 운영 가이드

## 1) 현재 구성 개요
이 저장소는 다음 흐름으로 동작합니다.

1. `market_report.py`가 전일 시장 데이터를 수집합니다.
2. HTML 리포트를 생성해 `reports/latest.html`, `reports/market_YYYY-MM-DD.html`로 저장합니다.
3. GitHub Actions(`.github/workflows/daily-market-report.yml`)가 매일 UTC 01:00(한국시간 10:00)에 자동 실행됩니다.
4. 리포트 변경이 있으면 자동 커밋/푸시를 시도합니다.
5. SMTP 시크릿이 설정되어 있으면 이메일로 리포트를 발송합니다.

---

## 2) main 브랜치 기준 필수 파일
아래 파일이 모두 있어야 정상 운영 가능합니다.

- `.github/workflows/daily-market-report.yml`
- `market_report.py`
- `reports/latest.html`
- `reports/market_2026-04-08.html` (예시 초기 리포트)

> 참고: 날짜 리포트 파일은 실행 날짜에 따라 계속 추가됩니다.

---

## 3) 다음 진행 순서 (실무 권장)

### Step A. GitHub Secrets 설정
저장소 `Settings > Secrets and variables > Actions`에서 아래를 등록합니다.

- `SMTP_SERVER`
- `SMTP_PORT`
- `SMTP_USERNAME`
- `SMTP_PASSWORD`
- `MARKET_REPORT_TO`
- (선택) `MARKET_REPORT_FROM`

### Step B. 수동 1회 실행
`Actions > Daily Market Report > Run workflow`로 수동 실행합니다.

확인 포인트:
- workflow가 성공 상태인지
- `daily-market-report` artifact가 생성되는지
- 이메일이 실제 수신되는지

### Step C. 정기 실행 모니터링
스케줄 실행 후 아래를 주기적으로 확인합니다.

- 실패한 job 여부
- `reports/latest.html` 갱신 여부
- 장기간 누적 파일(`reports/market_*.html`) 관리 필요 여부

---

## 4) 자주 발생하는 이슈

### 4-1. PR에서 conflict 발생
`market_report.py` 충돌이 나면 PR 화면에서 `Resolve conflicts`로 정리 후 `Commit merge`를 수행합니다.

### 4-2. 외부 데이터 소스 장애
일부 심볼 조회 실패 시 경고가 리포트에 표시되며, 가능한 데이터만으로 리포트 생성이 계속됩니다.

### 4-3. GitHub Actions에서 push 실패
권한/브랜치 보호 정책으로 push가 막히면 다음 중 하나를 적용합니다.

- Actions 권한(`contents: write`) 재확인
- 브랜치 보호 규칙에서 bot push 정책 확인
- 자동 커밋 대신 artifact 저장 방식으로 전환

---

## 5) 운영 개선 아이디어

1. 리포트 보관 기간 정책(예: 최근 90일) 적용
2. 메일 본문 요약(상승/하락 상위 지표) 추가
3. 실패 알림(Slack/Discord/Webhook) 연동
4. 테스트 코드 추가(HTML 생성 함수, 날짜 계산 함수)
